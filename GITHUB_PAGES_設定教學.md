# GitHub 新專案與 GitHub Pages 設定步驟

## 一、建立新的 GitHub 專案
1. 登入 GitHub。
2. 右上角按 `+` → `New repository`。
3. Repository name 建議填：`cpas-beginner-simulation`。
4. 選擇 `Public`。
5. 不需要勾選 README、.gitignore、license，因為本資料夾已經有檔案。
6. 按 `Create repository`。

## 二、上傳檔案
1. 進入剛建立的 repository。
2. 點選 `uploading an existing file`。
3. 把本資料夾中的 `index.html`、`style.css`、`data.js`、`app.js`、`README.md` 全部拖曳上傳。
4. Commit message 可填：`first version of CPAS beginner simulation`。
5. 按 `Commit changes`。

## 三、開啟 GitHub Pages
1. 進入 repository 的 `Settings`。
2. 左側選單點 `Pages`。
3. Source 選 `Deploy from a branch`。
4. Branch 選 `main`，資料夾選 `/root`。
5. 按 `Save`。
6. 等候約 1-3 分鐘，GitHub 會產生網址。
7. 之後課堂可直接分享這個網址給學員。

## 四、之後如何修改題目或個案回答
1. 打開 repository 裡的 `data.js`。
2. 按右上角鉛筆圖示編輯。
3. 修改 `stage1` 裡面的題組、回答、可透露資料。
4. 修改完成後按 `Commit changes`。
5. GitHub Pages 會自動更新。

## 五、建議課堂操作
- 第一輪：請學員不要看 CPAS，專心提問與蒐證。
- 第二輪：按下「揭示 CPAS 對照資料」。
- 第三輪：請學員回頭檢查資料格，判斷哪些資料可以支持 CPAS 分數。
- 第四輪：進入報告骨架頁，整理出初級版報告。
